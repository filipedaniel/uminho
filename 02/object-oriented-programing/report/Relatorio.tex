\begin{abstract}
O projecto da unidade curricular Programação Orientada aos Objectos consiste no desenvolvimento de uma aplicação que permita registar e simular actividades desportivas chamada FitnessUM. O processo de implementação do mesmo encontra-se devidamente detalhado neste relatório.
\end{abstract}

\section{Introdução}
\paragraph{}O tema deste projecto era o desenvolvimento de uma aplicação de simulação de actividades desportivas. De forma a conseguir isso o grupo recorreu ao ambiente de desenvolvimento BlueJ. Este relatório descreve detalhadamente como se encontra organizado esta mesma aplicação, para isso encontra-se dividido em seis fases descrevendo cada uma delas uma etapa do projecto. 
\\\\
A primeira fase do projecto consiste na implementação e organização das actividades desportivas, de modo a ser fácil a expansão das mesmas futuramente.
\\\\
 A segunda fase descreve a implementação da classe \textit{Amigo} e sua organização, sendo esta futuramente utilizada pela classe \textit{Utilizador} .
\\\\
A terceira fase descreve a classe mais importante, a classe \textit{Utilizador}, esta utilizando as duas classes criadas nas duas primeiras fases.
\\\\
 A quarta fase consiste em informação relacionada com estatística e  records de um dado utilizador.
\\\\
Na quinta fase encontrasse todos os menus de interface com o utilizador presentes nesta aplicação.
\\\\
Por ultimo, a sexta fase mostra as restantes classes criadas na realização desta aplicação que não foram abordadas anteriormente.

\section{Actividades}
\subsection{Organização das Actividades}


\begin{figure}[ht]
\centering
\includegraphics[width=1.0\textwidth]{actividades.png}
\caption{Organização das Actividades}
\end{figure}


Como mostra na figura 2, a classe \textit{Actividade} esta declarada com \textit{<abstract>} tendo como objectivo a reutilização do código através de herança. As actividades estão divididas em outras quatro classes, \textit{Outdoor} , \textit{Coletivas}, \textit{Aquatica} e \textit{Fitness} , também estas declaradas como classes abstractas pelo mesmo objectivo que a anterior. Cada uma destas subclasses têm actividades desportivas associadas, sendo que com este tipo de organização é possível inserir outras actividades facilmente devido mais uma vez à herança de classes. Nesta figura é possível ver também a classe \textit{Kilometragem} declarada como \textit{<Interface>} tendo como objectivo obrigar a implementação de um método que devolve o total de quilômetros nas actividades, neste caso, todas do tipo \textit{Outdoor}.  Existe também uma classe \textit{Actividades}, esta classe foi criada essencialmente para a organização de código, contendo um conjunto de actividades organizadas de uma certa forma.  Alem dessa existe mais duas classes \textit{ActividadeEnexException} e \textit{ActividadeExistException}, sendo cada uma delas exceções para funções implementadas na classe \textit{Actividades}.  

\subsection{Estrutura Escolhida}

\paragraph{}A estrutura escolhida para guardar a informação de cada actividade foi um \textit{HashMap}:
\begin{verbatim}
private HashMap<Integer,Actividade> l_actividades; 
\end{verbatim}
esta estrutura é declarada na classe \textit{Actividades}, escolhida pela eficiência, sendo a chave de inserção um identificador de cada actividade e como objecto a informação da actividade. Esta estrutura nesta situação é bastante eficiente devido ao facto de existir um identificador único para cada actividade.

\section{Estatística / Records}
	\subsection{Estatística}

\paragraph{}O menu que lista as estatísticas será acedido a partir do menu principal, apresentado mais a frente. Ao lista as estatísticas será possível aceder a três opções:
\begin{itemize}
	\item Estatísticas gerais total;
	\item Estatísticas gerais ultimo mês;
	\item Estatísticas gerais Anual;
\end{itemize}
cada uma delas mostra a mesma quantidade de informação, a primeira opção tal como o nome indica, mostra as estatísticas de todas as actividades já praticadas, a segunda do último mês e a terceira opção, pedindo ao utilizador um ano, mostra a respectiva estatística desse mesmo ano. 
\begin{itemize}
	\item Cada uma das opções mostrará:
	\begin{itemize}
		\item Numero total de actividades praticadas;
		\item Duração total despendidas em todas as actividades;
		\item Total de Calorias queimadas em todas as actividades;
		\item Total de quilômetros em todas as actividades (só contribui para este item as actividades que respondem a classe \textit{Kilometragem} já referido em anteriormente);
	\end{itemize}
\end{itemize}

\subsection{Recordes}
\paragraph{}O menu de recordes tal como o de estatística será acedido do menu principal. Este menu acaba por ser parecido com o de estatística, tratando-se apenas de uma lista de records simples. Ao listar será possível aceder a uma única opção:
\begin{itemize}
	\item Recordes Gerais
	\begin{itemize}
		\item Lista total de actividades praticadas;
		\item Máxima duração numa actividade física;
		\item Máximo de calorias queimas em uma só actividade; 
	\end{itemize}
\end{itemize}
a opção apresenta três resultados relacionados as actividades realizadoras por um dado utilizador.

\section{Conclusão}

\paragraph{}Numa primeira abordagem ao projecto tentamos definir classes genéricas para a implementação de actividades físicas, acabando por perceber que não era o mais correcto, a partir dai durante todo o resto do  desenvolvimento do trabalho, as nossas principais preocupações foram de definir as variáveis de instância como sendo o mais genérico possível possibilitando um maior leque de possibilidades na definição dos vários métodos e definir classes abstractas que no futuro permitissem implementar/acrescentar novas actividades facilmente, sem ser necessário a redefinição do código existente.
\paragraph{}É de salientar que além das estruturas de dados apresentadas neste relatório, foram usadas também \textit{TreeMap} e \textit{TreeSet} essencialmente para métodos de ordenação, permitindo logo à partida, por comparadores definidos nas classes, que é o caso das classes \textit{Actividade} e \textit{Data}, mas também, estando já definido, o comparador de \textit{Strings} uma ordenação na inserção.
